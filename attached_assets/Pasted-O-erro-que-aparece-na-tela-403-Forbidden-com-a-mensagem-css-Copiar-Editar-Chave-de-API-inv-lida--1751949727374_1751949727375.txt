Chave de API inválida ou sem permissão. Verifique se a chave está correta.
BotConversa API error: 403 Forbidden - {"error_message":"Api key is not valid"}
significa que o sistema está tentando autenticar na API do BotConversa com uma chave que não é válida, está mal formatada, ou pertence a um projeto sem permissão para esse endpoint.

✅ Como resolver isso passo a passo:
1. Verifique se está usando a chave certa para o ambiente
Você informou duas chaves:

e267dba7-1022-4f9f-a36b-b4be4b9a046e → Suporte

47c8c187-deb4-45ab-857a-df4af7ba1413 → Comercial

Confirme se a chave que está sendo usada corresponde ao ambiente da conta correta.
Se a tela onde o erro ocorreu for do comercial, use a segunda chave.

2. Verifique o formato da chave
As chaves do BotConversa são UUIDs (formato xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx). Certifique-se de que:

Não haja espaços extras antes ou depois da chave.

A chave esteja sendo enviada no header x-api-key, como indicado na documentação oficial.

Exemplo correto:

http

Editar
GET /v1/atendimentos HTTP/1.1
Host: backend.botconversa.com.br
x-api-key: 47c8c187-deb4-45ab-857a-df4af7ba1413
3. Verifique o endpoint usado
Certifique-se de que está acessando https://backend.botconversa.com.br e não uma URL errada ou ambiente de desenvolvimento. O erro 502 Bad Gateway vindo do Replit também pode indicar que o backend está tentando buscar algo da forma errada.

4. Teste a chave com um cliente HTTP (como Insomnia ou Postman)
Você pode testar manualmente a chave fazendo uma requisição GET para, por exemplo:

bash

Editar
GET https://backend.botconversa.com.br/v1/atendimentos
Headers:
  x-api-key: [cole aqui a chave]
Se retornar 200 OK, a chave é válida. Se retornar 403, a chave está errada ou com permissão insuficiente.

⚠️ Importante
Caso você tenha certeza de que:

Está usando a chave correta

O header está bem formatado

O endpoint é o correto

…e mesmo assim continua dando erro, então você pode:

✅ Regenerar a chave na conta do BotConversa
ou
✅ Entrar em contato com o suporte do BotConversa e informar o erro 403 com o requestId (se disponível no header de resposta).
